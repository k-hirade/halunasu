# レセプトチェッカー(病院向けレセプト点検システム)

## Halunasu STG版の扱い

このディレクトリは `../recept-checker` をHalunasu STGで試すために取り込んだコピーです。
Cloud Runでは `HALUNASU_ENV=stg` のときだけ利用し、UKEをアップロードしてその場で点検します。
履歴DBへの保存、点検除外の永続化、返戻・査定データの蓄積は行いません。
アクセス制限はパスワードではなく `RECEPT_CHECKER_ALLOWED_IPS` または `STG_GATE_ALLOWED_IPS` のCIDR/IP allowlistで行います。
STGでallowlistが未設定の場合はfail-closed（HTTP 503）にします。

医科レセプト電算処理システムの **UKEファイル** を解析し、審査支払機関(社会保険診療報酬
支払基金・国保連合会)の審査で返戻・査定されやすいポイントを **請求前に自動点検** する
ツールです。市販レセプトチェックソフト(べてらん君、Mighty Checker、レセプトチェッカーLS等)
の中核チェックを参考に、**公的に無償公開されている機械可読チェックデータ** をそのまま
判定基盤として使用します。

## 特徴

- **決定論的ルールエンジン**: すべての指摘に根拠(電子点数表・チェックマスタ・告示通知)を明示
- **令和8年度(2026年度)診療報酬改定対応**: 年度版マスター(令和6年度版/令和8年度版)を
  レセプトの**診療年月で自動切替**。改定で廃止されたコードの算定(例: 医療情報取得加算・
  医療DX推進体制整備加算のR8.6以降の請求)や**旧点数のままの請求**(F査定=固定点数誤り)を検出
- **公的データをフル活用**(1年度版あたり約160万行):
  - 基本マスター(医科診療行為 R6:10,192/R8:11,829 / 医薬品 / 傷病名 27,684 / 修飾語 / コメント / 特定器材)
  - 医科電子点数表: **背反テーブル4種**(1日/同一月/同時/1週間)77,494行、**算定回数テーブル** 5,928行、包括テーブル 248,235行
  - 支払基金コンピュータチェックマスタ: **医薬品適応・投与量・投与日数** 1,234,290行、
    **診療行為適応** 218,375行、**併用禁忌** 4,208行、**傷病名禁忌**、投与量グループ、対象事例 44,635件
- **縦覧点検対応**: コピー元は過去月レセプトをSQLiteに蓄積して複数月ルールを判定できます。
  Halunasu STG版ではDB保存を行わないため、過去月履歴が必要な縦覧点検は対象外です。
- **返戻・査定データの取込み分析**: コピー元は返戻ファイル(RECEIPTC.HEN/SAH)と増減点連絡書CSVを取込めます。
  Halunasu STG版ではこの画面を無効化しています。
- **病名整理・コード化支援**: 当月の診療内容と対応しない不要病名の抽出、
  未コード化傷病名(0000999)の「接頭語+基本病名+接尾語」分解によるコード化候補提示
- **ルールON/OFF**: ルール単位の有効/無効を画面から切替可能。
  Halunasu STG版ではプロセス内の一時設定で、DBへ永続化しません。
- **施設ルール**: CSVで自院独自のチェック(適応・背反・回数・算定もれ・コメント要件)を追加可能
- **Web UI + CLI + Excel/CSVレポート**

## セットアップ

```bash
python3 -m venv .venv
.venv/bin/pip install -r requirements.txt

# 1) 公的マスターのダウンロード(年度版ごと)
.venv/bin/python -m receipt_checker.masters.download --version R06 --dest master_data/raw_r06
.venv/bin/python -m receipt_checker.masters.download --version R08 --dest master_data/raw_r08

# 2) SQLiteデータベースの構築(年度版ごと・各約190MB)
#    レセプトの診療年月に応じて適用版が自動選択されます(改定またぎ対応)
.venv/bin/python -m receipt_checker.masters.official_import \
    --raw master_data/raw_r06 --db master_data/masters_R06.db --label 令和6年度版
.venv/bin/python -m receipt_checker.masters.official_import \
    --raw master_data/raw_r08 --db master_data/masters_R08.db --effective 202606 --label 令和8年度版
```

単一年度版だけで運用する場合は `--db master_data/masters.db` の1本でも動作します。
公的マスターDBがない場合も、同梱のフォールバックCSV(`receipt_checker/masters/data/`)で
基本的な点検は動作します(適応・背反・回数チェックの網羅性は大きく下がります)。

## 使い方

### Web UI

```bash
RECEPT_CHECKER_ALLOW_NON_STG=true .venv/bin/python run.py          # → http://127.0.0.1:8230
```

1. トップページでUKEファイルをアップロード → 自動点検
2. 結果画面: 重大度(エラー/警告/情報)・分類・レセプト番号でフィルター、レセプト詳細表示
3. 「Excel出力」で医事課の点検作業用ブックを出力
4. Halunasu STG版では履歴保存を行いません。縦覧点検は過去月DBなしで実行される範囲に限定されます。

### コマンドライン

```bash
.venv/bin/python check_cli.py レセプト.UKE                # 画面に結果表示
.venv/bin/python check_cli.py レセプト.UKE -o 結果.xlsx   # Excelレポート
```

### サンプルデータで試す

```bash
# その場でUKEを点検
.venv/bin/python check_cli.py sample_data/sample_202406.UKE

# 令和8年度改定またぎのデモ(廃止コード・旧点数の検出)
.venv/bin/python check_cli.py sample_data/sample_202606.UKE
```

サンプルには実在コードで以下のエラーが埋め込まれています:
特定疾患療養管理料の月2回超過/特定疾患処方管理加算の重複(背反)/高血圧への特定疾患療養管理料
(令和6年改定で対象外)/1日最大量・同一成分合算の過量/向精神薬30日制限超過/初診日の外来管理加算/
初診料への包括/処置と外来管理加算の同日算定/併用禁忌(公的+施設ルール)/禁忌傷病名への投与/
疑い病名の長期放置/初診料の再算定(縦覧)/PSAの患者当り1回超過(縦覧)/検査判断料の算定もれ/
性別不一致病名/傷病名なしレセプト など

## 点検ルール一覧(12カテゴリ・40超)

| 分類 | ルール例 | 主なデータソース |
|---|---|---|
| 形式 | 必須項目・日付整合・マスター未登録コード・実日数・合計点数検算 | 記録条件仕様/基本マスター |
| 患者属性 | 性別固有の傷病名・診療行為、年齢制限 | 基本マスター(上下限年齢) |
| 傷病名 | 疑い病名の長期放置・重複・主傷病・未コード化・転帰・病名なし | 傷病名マスター |
| 医薬品適応 | 適応病名もれ(病名もれ)・禁忌傷病名・性別年齢適応 | チェックマスタ(適応123万行) |
| 診療行為適応 | 検査・処置・管理料の適応病名もれ(入外・疑い可否条件付き) | チェックマスタ(21.8万行) |
| 用量・日数 | 1日最大量・最長投与日数・同一成分合算・湿布63枚 | チェックマスタ+施設ルール |
| 併用禁忌 | 注射薬の併用禁忌(公的)+内服等の施設ルール(一般名) | チェックマスタ+CSV |
| 併算定(背反) | 1日/同月/同時/週の背反・包括項目・加算の単独算定 | 電子点数表+注加算コード |
| 回数制限 | 日・週・月・初診時・患者当り等の上限、実日数超過、初診と再診の同日算定 | 電子点数表・算定回数テーブル |
| 算定もれ | 検査判断料もれ・基本診療料なし・処方料もれ・施設パターン | 基本マスター(判断区分)+CSV |
| 摘要記載 | コメント必須項目・コメントコード実在性・自由記載の多用 | コメントマスタ+CSV |
| 縦覧点検 | ◯月に1回・患者当り1回の通算、初診料再算定、疑い病名持ち越し | 電子点数表+履歴DB |
| 病名整理 | 当月の診療内容と対応しない不要病名の抽出(レセプト病名対策) | チェックマスタ逆引き |
| 改定対応 | 廃止コードの算定・固定点数(旧点数)の不一致 | 年度版マスター突合 |

ルールの有効/無効は「チェックルール一覧」画面のチェックボックス、または
`CheckEngine(enabled_rule_ids=..., disabled_rule_ids=...)` で制御できます。
Halunasu STG版では個別指摘の除外保存は行いません。

## コピー元で提供される返戻・査定データ分析

コピー元アプリでは、オンライン請求システムから取得できる以下のファイルを取込むと、
事由別(A適応外/B過剰/D要件不合致/F固定点数誤り…)・診療区分別・月別に集計されます:

- **返戻ファイル**: RECEIPTC.HEN(一次請求返戻)/ RECEIPTC.SAH(再審査等返戻)
- **増減点連絡書CSV**: RIzogn…形式(毎月5日頃からダウンロード可能)

Halunasu STG版ではこの機能はメニューから外しています。必要になった段階で、DB非永続ではなく保存先を設計してから有効化します。

## 施設ルールのカスタマイズ

`receipt_checker/masters/data/` のCSVを編集(UTF-8・ヘッダー行あり):

| ファイル | 内容 |
|---|---|
| `drug_indications.csv` | 医薬品→適応病名(コード;区切り+キーワード) |
| `act_indications.csv` | 診療行為→適応病名 |
| `contraindications.csv` | 併用禁忌(一般名レベル。内服の禁忌はここで拡張) |
| `exclusive_pairs.csv` | 併算定不可(same_day/same_month) |
| `frequency_limits.csv` | 回数制限(day/week/month/Nmonths) |
| `missing_patterns.csv` | 算定もれパターン(トリガー→期待算定) |
| `comment_requirements.csv` | コメント・症状詳記の必須項目 |
| `age_sex_restrictions.csv` | 年齢・性別制限 |
| `shinryo_koi.csv` ほか | 公的DBがない場合の名称解決用フォールバック |

## データの出所(すべて公的・無償)

| データ | 提供元 | 更新 |
|---|---|---|
| 基本マスター | 診療報酬情報提供サービス(shinryohoshu.mhlw.go.jp)/支払基金 | 随時(改定期は年度版切替) |
| 医科電子点数表 | 社会保険診療報酬支払基金(ssk.or.jp) | 随時 |
| コンピュータチェックマスタ | 社会保険診療報酬支払基金「コンピュータチェックに関する公開」 | 半期ごと目安 |

診療報酬改定・マスター更新時は `download.py` のURL(年度版)を確認のうえ、
ダウンロード→取込みを再実行してください。

## テスト

```bash
.venv/bin/python -B -c "import ast, pathlib; [ast.parse(path.read_text()) for path in pathlib.Path('services/recept-checker').rglob('*.py')]"
```

元リポジトリのpytest一式は `../recept-checker/tests` に残しています。Halunasu STG版はDB非永続化しているため、コピー元のDB前提テストは同梱していません。

## 重要な注意(免責)

- 本ツールの指摘は**請求前の院内点検を支援する参考情報**です。算定の最終判断は、
  告示・通知・疑義解釈および審査支払機関の取扱いに基づき、医事課・診療部門で行ってください。
- 支払基金の公開資料にも「コンピュータチェック対象事例の診療内容の適否は、画一的・一律的に
  適用するものではなく、個別症例ごとに審査委員会の医学(薬学)的判断により決定」と明記されています。
- 指摘がゼロでも返戻・査定が発生しないことを保証するものではありません(施設基準・届出状況・
  症状詳記の内容等は点検対象外です)。
- 患者情報を含むUKEファイルはすべてローカルで処理され、外部送信されません。

## アーキテクチャ

```
receipt_checker/
├── parser/uke_parser.py   UKE(レセ電CSV・Shift_JIS)→ データモデル
├── models.py              Receipt / Disease / ServiceItem / Finding
├── masters/
│   ├── download.py        公的データのダウンロード
│   ├── official_import.py 公的データ → SQLite(masters.db)
│   ├── official.py        公的マスターのクエリ層
│   ├── loader.py          MasterSet(公的DB+施設ルールCSV)
│   └── data/*.csv         施設ルール+フォールバックマスター
├── rules/r01〜r12         点検ルール(12カテゴリ)
├── engine.py              チェックエンジン(全ルール×全レセプト)
├── store.py               縦覧点検用の履歴DB(SQLite)
├── report/export.py       Excel/CSVレポート
└── web/                   FastAPI + Jinja2 UI
```
