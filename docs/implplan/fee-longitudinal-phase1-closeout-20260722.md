## L1. [P1] 空抽出ガード(0イベント検証駆動リトライ)

### 意図

全文抽出が空(イベント0件)を返す単発の取りこぼしは、メモとは無関係に本番でも起こる。
既存の検証駆動リトライ(line_review欠落→欠落行だけ再抽出)は「line_reviewが全行false」の
完全空振りを検出できない(照合上は完全なため)。決定論レーンが「本文に算定シグナルがある」と
言っているのに抽出が空なら、それは矛盾であり1回だけ再抽出する価値がある。

### 仕様

対象: `services/fee-api/src/clinical-calculation-input.js`(line_review照合と同じ層)。

1. トリガー(**全て決定論**。以下のいずれかが今回本文の肯定節に存在し、かつ
   `clinical_events` が0件):
   - 否定・予定・過去等を除外した辞書スキャン候補が1件以上ある
   - `session.orders` の具体的な名称が今回本文にも記載されている
   - `session.diagnoses` の病名が今回本文にも記載され、抽出 `diagnoses` が0件
   - (メモ利用時)直近スナップショットに has_billable_act=true の行が存在した
2. アクション: **全文再抽出を1回だけ**実行し、既存の和集合規約で統合する。
   再抽出後も空なら確認事項を出す:
   「AI抽出が空でした。決定論候補(辞書・病名・施設ルール)のみ提示しています。
   本文に算定対象の行為がないか確認してください。」(黙って空のまま進めない)
3. アプリ層の意味的リトライは上限1回(`line_review` 欠落時の再抽出と合算し、
   過剰呼出にならないことをテストで固定)。不正形式に対するprovider/SDK内部の
   制御は別レイヤーとして扱う。
4. 観測: metrics `emptyExtractionGuard: {triggered, reasonCodes, retryAttempted, recovered}` / trace stage
   `empty_extraction_guard`。評価スクリプトにも記録。
5. ゲート: `FEE_EMPTY_EXTRACTION_RETRY`。メモ機能とは独立し、STGは既定true、PRODは既定false。

### 実装結果

- `clinical-calculation-input.js` に肯定文脈限定の矛盾判定、全文再抽出1回、未回復issueを実装。
- 空抽出と `line_review` 欠落は共通のapplication semantic retry budgetを使い、合計1回で停止。
- `server.js` のperformance snapshotへ抽出モード・理由・回復成否を保存し、Cloud Runログでも確認可能。
- 計測ログには理由コードと件数だけを残し、カルテ本文の根拠断片は含めない。
- 肯定辞書一致、否定反例、2回連続空、ネストしたcache token集計のテストを追加済み。

### テスト

- 空抽出+辞書ヒットあり→1回再抽出し、2回目の結果が統合される。
- 空抽出+シグナル無し(本当に算定なしの本文)→再抽出しない(平文の相談記録等で無駄打ちしない)。
- 2回連続空→確認事項が出る。リトライは合計1回で止まる。
- gold 2系統・反例コーパス不変。
