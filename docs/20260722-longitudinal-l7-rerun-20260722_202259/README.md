## 結論

`1002 / 1004 / 1006 / 1010 / 1011`を各3反復、1反復4カルテの時系列でSTGへ投入した。
受診区分と同一建物区分を受診単位で入力し、施設ルールで生成された明細を確認・承認したうえで
患者×月のレセプト案を既存UKEと比較した。合計60算定は同一revision
`fee-api-stg-00169-6qx`で完走した。

M6のハーネス修正効果とL7本来の受入条件は合格と判断する。

- 19訪問を`home_visit`、1002の電話再診1件を`outpatient`として処理した。
- 施設患者は同一建物3〜6人、1006は個人宅として決定論入力された。
- 同一建物区分の未確定候補は0件。施設患者では`114030310 / 180726010`、個人宅では
  `114001110 / 180725910`が全訪問で選択された。
- 月次確定点数は全患者で非0。既存UKEとの確定一致は各患者2〜3コード、候補込みでは3〜4コード。
- `memoUsedVisitCount=0`、`historyUnavailableCount=0`。全行置換mockではメモ非関与が再確認された。
- 初回L7 RunよりUKE検知は全患者で改善し、旧正条件Run比でも非悪化だった。

したがって縦断コンテキストPhase 1はクローズ可能である。ただしPRODの
`FEE_EXTRACTION_MEMO`有効化は、親計画どおり顧客カルテの実copy-forward率を測るまで保留する。

電話再診コード未検知と全文LLM抽出の揺れはL7とは分離した既知課題として残る。

## M3: 空抽出ガード

専用1011 Runは[結果](./1011-guard/result.json)を参照する。12受診中、ガード発火は0件だった。
6月27日の本文は反復ごとにイベント`0 / 4 / 0`だったが、記載は
「胃瘻・吸引管理を継続」という管理方針であり、当日の喀痰吸引実施記録ではない。

この本文は次の決定論トリガーを満たさない。

- マスタ名`喀痰吸引`の肯定的な完全一致ではない。
- 当日orderとして喀痰吸引を入力していない。
- 抽出病名は存在し、病名0件矛盾ではない。
- 全行置換のため継続billable行ではない。

ガードを「吸引管理」へ広げると、実施していない管理方針を手技実施と誤認するため変更しない。
別反復の4イベントはLLMが管理記載を広くイベント化した結果であり、むしろ過抽出側の揺れである。

正式5患者Runでもガード発火は0/60だった。これはガード無効ではなく、今回のイベント0受診に
肯定的な決定論シグナルが無かったことを表す。

## M4: 電話再診

1002の6月16日は`outpatient`として処理され、訪問診療料の誤追加は止まった。
ただし既存UKEの`112007950`電話等再診料は候補・確定のどちらにも出ず、通常の
`112007410`再診料と`180820010`物価対応料が確定した。`112015770`明細書発行体制等加算は一致した。

原因は今回のM1が大区分`setting=outpatient`までで、H3の
`encounterDetails.visitKind=telephone_revisit`と電話等再診料候補化が未実装なためである。
L7では既知差分として扱い、H3で対応する。

## M5: メモ無効対照

60/60受診で`memoUsed=false`、`memoHitLineRatio=0`、抽出モードは`full`だった。
この全行置換データではメモ有効化が算定結果へ関与していないため、メモ無効対照Runは実施しない。
メモ非関与の観測そのものを無害性の根拠とする。
