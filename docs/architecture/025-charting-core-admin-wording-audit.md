# カルテ作成・施設管理画面 文言監査レポート

作成日: 2026-05-31

## 目的

カルテ作成アプリと施設管理画面に表示される文言を、IT知識の少ない医療機関ユーザーにも伝わる表現へ寄せるためのASIS/TOBEを整理する。

今回の調査は実装修正前の文言監査であり、対象は以下に限定する。

- カルテ作成: `apps/charting-web/components/session-launcher.js`
- カルテ作成: `apps/charting-web/components/encounter-workspace.js`
- カルテ作成の設定画面: `apps/charting-web/components/admin-console.js`
- 施設管理画面: `apps/core-admin/index.html`

LP、診療報酬算定、紹介状作成、テストコード内の文言は今回の一次対象外とする。

## 推奨する用語統一

| ASIS | TOBE | 理由 |
| --- | --- | --- |
| 診療セッション | 診療一覧 / 診療記録 | 「セッション」はIT寄り。画面タイトルは「診療一覧」、個別画面は「診療記録」が自然。 |
| セッション履歴 | 過去の診療 | 一般語にする。 |
| Core患者 / Core施設 | 登録済み患者 / 登録済み施設 | Coreは内部名。ユーザーには不要。 |
| 共通マスター | 病院共通データ | 「マスター」は業務アプリ慣れしていないと伝わりづらい。 |
| 患者インデックス | 患者名簿 | インデックスは内部データ構造。 |
| プロダクト契約 | アプリ利用設定 | 契約状態を扱う画面だが、現場では「どのアプリが使えるか」が主眼。 |
| 開示請求 | 個人情報の依頼 | 法務用語として残す場合も、タブ名は意味が広い表現にする。 |
| 監査イベント / 監査ログ | 操作履歴 | 監査は管理者向け詳細語。通常画面では操作履歴で十分。 |
| 組織 | 病院 | 現在のユーザー単位に合わせる。 |
| メンバー | 職員 | 医療機関の管理画面として自然。 |
| ロール | 権限 | ロールは内部名。 |
| ログインセッション | ログイン状態 | 失効などの概念を説明しやすい。 |
| 失効 | 強制ログアウト | 操作結果が明確。 |
| MFA | 2段階認証 | 既に一部修正済み。カルテ設定画面にはまだ残っている。 |
| PC / この端末 / パソコン | このパソコン | 文言を統一する。 |
| iPhone / スマホ | スマホ | iPhone限定でない限りスマホに統一。QR読み取り説明では「スマホのカメラ」。 |

## 実装時の注意

- 「SOAP」と「プロンプト」は機能名として残す。周辺文言だけを、`プロンプト名`、`プロンプト本文`、`病院全体で最初に使うプロンプト`、`職員を指定` のように具体化する。
- 施設管理画面の `プロダクト契約` は課金設計と関連するが、UIの第一階層は `アプリ利用設定` の方がユーザーの目的に近い。
- `削除` は医療データでは強い言葉なので、実削除でない場合は必ず `非表示`、`一覧から外す`、`破棄` のように操作実態へ合わせる。
- 文言変更はE2Eのセレクタに影響する。ボタン名・見出し名で取得しているテストは同時更新が必要。
