# 候補安定化 T6 STG再計測（5患者）

## 結論

患者1001、1004、1006、1007、1012を、STGで各3回測定した。

評価経路は次のとおりである。

`カルテ本文 -> セッション算定 -> 患者 x 月の月次レセプト案 -> 既存UKE比較`

5患者、15患者Run、45算定はAPIエラーなく完走し、45算定すべてでOpenAI抽出が使われた。
確定月次点数は5/5患者で3回同一だった。一方、月次候補集合が3回同一だったのは2/5患者で、
T6目標の5/5には達していない。

ただし、この結果をT1〜T5のデプロイ後評価として扱ってはいけない。応答内容から、計測時のSTGには
T1/T2/T4/T5が反映されていないことを確認した。今回の測定は現行STGの基準値として保存し、
T1〜T5をデプロイした後に同じ条件で再実行する必要がある。

## デプロイ未反映の根拠

1. **T1/T2未反映**: 1007で、同じ`codeCandidates = [114003710, 114004110]`を持つ在宅酸素候補が
   2行のまま残り、コード未確定候補に2,400点が表示された。T1/T2反映後は1行へ統合され、0点になる。
2. **T4未反映**: `113001810 特定疾患療養管理料`が、1006では反復2のみ、1007では反復1・2のみ、
   1012では反復2・3のみに出た。病名駆動レーン反映後の決定論的な挙動ではない。
3. **T4計測値なし**: 45算定すべてで`diseaseIndicationScan`が応答になく、評価結果では0ms・候補0件になった。
4. **T5未反映**: 全15 Runで`monthlyMetrics`と`baselineDiagnosisMetrics`が`null`だった。
5. **T3**: 今回は全患者を`home_visit`で実行したため、外来再診専用の外来管理加算候補では反映有無を判定できない。
