## 結論

STGの同一revision `fee-api-stg-00166-7zk` で5ケースを計測した。全文抽出対照は各ケース3回、
合計15回実行し、確定明細・候補集合の対照内揺れは0件だった。

- 完全copy-forward: 別セッション参照と同一セッション再計算の双方で全文対照と一致した。
  `memoHitLineRatio=1`、新規/消失行0、OpenAI呼出し0だった。
- P行だけ変更: 継続3・新規1・消失1で、全文対照と一致した。
- 行為消失: 前回だけの創傷処置は現在の明細・候補に混入せず、`removedLineCount=1` と
  `extraction_memo` traceの双方に記録された。
- 全行変更: メモを使わず全文抽出へ戻り、全文対照と一致した。
- 院外処方箋の新規記載: 既知の`visit_facts`引継ぎ制約が実害を生じた。全文対照は
  処方箋料系だった一方、メモ経路には院内調剤・処方料とアムロジピン32点が残り、
  確定明細完全一致に不合格となった。

この結果を受け、新規または消失行に院内/院外処方・処方箋・一般名処方・リフィルの語彙が
ある場合は行スコープ抽出を使わず、全文抽出へフォールバックする修正を実装した。
特定の患者、SOAP本文、薬剤名には依存しない。抽出スナップショットschemaはv2へ更新し、
旧snapshotは自動失効する。API再デプロイ後のSTG再計測は未実施である。
