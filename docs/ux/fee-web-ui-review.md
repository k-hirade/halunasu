# 診療報酬算定アプリ（fee-web）UIレビューと実装方針

作成日: 2026-06-08
対象: `apps/fee-web`（Next.js）／比較対象: `apps/charting-web`（SOAP作成）
レビュー範囲: 画面構成・操作導線・終端アクション・主要UI。現行コードに照らして、実装すべき内容だけに整理。

---

## 0. 結論

fee-web はカルテ本文から算定候補を作る骨格はできているが、操作導線がまだ「全部入りの作業画面」に寄っている。改善すべき中心は、余計な中継画面を消すこと、入力をカルテ本文中心に戻すこと、条件確認をポップアップへ逃がすこと、最後に「出力して終わる」導線を作ること。

残す方針:
- 増点提案カードは `条件を確認` に集約する。カード上で直接採用はしない。
- 採否はモーダルまたは明細行で `算定する / 保留 / 算定しない` に統一する。
- 保存ボタンは不要。入力と採否は自動保存する。

---

## 1. 実装する変更

### 1-1. `/sessions/new` の中継ページをなくす

現状:
- `/sessions/new` は空の算定記録を作るだけの中継ページ。
- 一覧の `算定を開始` でも同じことができる。

変更:
- `/sessions/new` は一覧へリダイレクトする。
- `NewFeeSessionView` は削除する。
- 新規作成は一覧の `算定を開始` に一本化する。

### 1-2. 終端アクションを作る

現状:
- `レセプト案` は表示だけで、作業の終わりがない。

変更:
- 下部アクションバーに `レセプト案をコピー` を追加する。
- レセプト案がない場合は無効化する。
- コピー後は完了メッセージを出す。

### 1-3. 条件・オーダー手動編集をポップアップにする

現状:
- 診療科、区分、診療日、請求月、病名、対応範囲、マスター検索、オーダー表が左カラムに常時表示され、カルテ本文の主導線を弱めている。

変更:
- 左カラムには患者、カルテ本文、条件サマリだけを表示する。
- `算定条件を確認` と `オーダーを確認` のボタンからポップアップを開く。
- ポップアップ内で診療条件、病名、対応範囲、マスター検索、オーダー表を編集する。

### 1-4. `?` ボタンを `詳細` に変更する

現状:
- 算定中明細の右端に `?` ボタンがある。

変更:
- `?` を `詳細` に変更する。
- 提案/確認カードの `条件を確認` と意味を揃える。

### 1-5. 旧レビューUIを削除する

方針:
- 現行UIは `CandidateWorkbench` に一本化する。
- 未使用の旧UI関数は残さない。
- 採否語彙は現行の `算定する / 保留 / 算定しない` に寄せる。

### 1-6. 自動保存表示を整理する

現状:
- 下部バー左側に成功メッセージや `入力内容を確認してください` と保存状態が混ざっている。

変更:
- 下部バー左側は `保存済み` / `保存中...` / `保存失敗` のみにする。
- メッセージは画面上部の通知に任せる。

## 2. 今回はやらないこと

- 提案カードに直接 `算定する +○点` は置かない。採用判断は `条件を確認` モーダルに集約する。
- 全アイコンの一括統一はしない。意味が伝わらない `?` だけ直す。
- レセ電などの正式出力は今回入れない。まずはレセプト案のコピーを終端アクションにする。
- ステッパー追加は今回は見送る。中継ページ削除、ポップアップ化、終端コピーを優先する。
