# 施設基準辞書

2026-05-18時点の実装では、地方厚生局の施設基準略称を算定ロジック内部の `rule_key` に正規化する辞書を `src/medical_fee_calculation/facility_standard_dictionary.py` に置く。

## 目的

地方厚生局データの施設基準は、`検Ⅱ`、`画２`、`遠画`、`一般入院` のような略称で入る。一方、算定ロジックでは点数コードや条件分岐を安定させるため、略称を直接比較せず、内部rule keyで扱う。

## 実装済みカテゴリ

| カテゴリ | 代表略称 | 内部rule key | 接続先 |
| --- | --- | --- | --- |
| 検体検査管理加算 | `検Ⅰ` / `検Ⅱ` / `検Ⅲ` / `検Ⅳ` | `lab_management_1` - `lab_management_4` | 検体検査管理加算候補 |
| 画像診断管理加算 | `画１` / `画２` / `画３` / `画４` | `image_diagnostic_management_1` - `image_diagnostic_management_4` | CT/MRI、単純系画像診断管理加算候補 |
| 遠隔画像診断 | `遠画` | `remote_image_diagnostic` | 遠隔画像診断管理加算候補 |
| 入院基本料系 | `一般入院`、`特定入院`、`専門入院`、`療養入院`、`精神入院`、`結核入院`、`障害入院`、`診入院`、`診療養入院` | `*_basic_inpatient` | Step6の入院基本料候補 |

## 正規化

`resolve_facility_standard_rule_key()` は、全角/半角、ローマ数字、算用数字の揺れを吸収する。例として、`検Ⅱ`、`検II`、`検2` はすべて `lab_management_2` に解決される。`画２` と `画2` は `image_diagnostic_management_2` に解決される。

複数施設基準から最上位を選ぶ場合は `select_facility_standard_rule_key()` を使う。検体検査管理加算と画像診断管理加算は、届出レベルの高いものを優先する。

## 算定ロジックへの接続

検体検査管理加算は、従来の略称直接比較をやめ、辞書で `lab_management_*` に変換してから対応コードを選ぶ。

画像診断では、`ImagingOrder` に次の明示フラグを追加した。

| フィールド | 意味 |
| --- | --- |
| `diagnostic_management_add_on` | 画像診断管理加算を候補化する |
| `remote_diagnostic_management_add_on` | 遠隔画像診断管理加算を候補化する |

CSVの日本語presetでは、`画像診断管理加算` / `画像診断管理`、`遠隔画像診断管理加算` / `遠隔画像診断` を上記フラグへmappingする。

対応済みコード。

| 条件 | コード |
| --- | --- |
| `画１` + 単純撮影/造影撮影/乳房撮影 | `170025210` |
| `画１` + CT/MRI | `170025510` |
| `画２` + CT/MRI | `170025710` |
| `画３` + CT/MRI | `170702410` |
| `画４` + CT/MRI | `170035810` |
| `遠画` + `画１` + 単純撮影/造影撮影/乳房撮影 | `170025810` |
| `遠画` + `画１` + CT/MRI | `170026110` |
| `遠画` + `画２` + CT/MRI | `170026310` |
| `遠画` + `画３` + CT/MRI | `170702810` |
| `遠画` + `画４` + CT/MRI | `170036010` |

施設基準が不足する場合、画像診断管理加算は自動候補化せず `imaging_fee needs_review` として返す。入院基本料でも、`facility_standard_key` または `ward_kind` がこの辞書で解決でき、医療機関profile側に該当rule keyがない場合は、入院基本料候補を出さず `inpatient_basic_fee needs_review` として返す。

## 制約

この辞書は、施設基準略称を算定ロジックに渡すための入口であり、施設基準の届出有効性そのものは地方厚生局 importer と `hospital_profile` が担う。入院基本料系のrule keyはStep6の入院基本料候補で使える。入院料コードの自動選択、病棟単位、DPC係数、入退院日からの日数計算は未実装である。
