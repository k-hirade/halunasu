# ORCA カルテ事例集（外来11例）— end-to-end 算定 gold

「**カルテ（病名・診療内容）→ 診療報酬算定（明細・点数）**」の対応が取れた、非PHIの公開素材。
診療報酬算定エンジン／抽出（カルテ本文→病名・オーダー）の**評価用 gold** として使える。
（既存の `gold-dataset-step7.md` は「構造化オーダー→算定」の gold。本データは**カルテ起点**まで遡れる点が新しい。本docには各事例の**カルテ本文（入力）／SOAP書き起こし版（入力）／算定明細（期待出力）**を収録。）

## 出典・ライセンス
- 出典：**ORCA Project / 日本医師会ORCA管理機構**「日医標準レセプトソフト 外来版 カルテ事例（ver 4.8.0 対応, 2016-10-28）」
  - 一覧：https://www.orca.med.or.jp/receipt/users/manual/attachments/karte.html
  - PDF：`https://ftp.orca.med.or.jp/pub/data/receipt/outline/karte/pdf/2016-10-28-karte-caseNN-print.pdf`（NN=01〜11、まとめは `-cases-all11.pdf`）
  - 取得スクリプト：`scripts/fetch_orca_karte_cases.sh`
- 各PDFは**「擬似カルテ（実カルテではない）＋それに対応する算定例（診療報酬明細）」**を含む。
- **PHIなし**（事例 一〜十一の合成患者）。
- ⚠ **再配布注意**：ORCA配布教材のため、本リポジトリには**PDF/本文そのものはコミットしない**方針。本docには評価に必要な**事実（病名・算定明細・点数）**のみ転記。外部公開時は配布条件を要確認。

## ⚠ 重要なcaveat（点数の鮮度）
- 点数は **ver 4.8.0（2016年）時点の診療報酬**。**その後の改定で点数・要件は変わっている**。
- よって本データは「**算定の構造・区分・組み合わせ・ルール**」の参照／回帰の土台として使い、**点数の絶対値は現行改定 or ORCAで再計算して照合**すること（点数そのものを固定 gold にしない）。

## ORCA側の前提（算定例はこの設定で生成）
算定例を再現・照合するための ORCA システム管理マスタ設定（PDF冒頭より）：
- 医療機関種別：**診療所** ／ 院外処方区分：**院内** ／ 構成：医師・看護師・事務員
- 診療科：内科・精神科・外科・整形外科・眼科
- 施設基準フラグ：`0023 0721 0755 3168`（明細書発行体制等加算 等）
- 外来初診・再診料／病名疾患区分からの自動発生／画像診断管理加算：**算定する**
- 労災・自賠責：点数単価 **12円**、加算率 **120%**、労災指定医療機関：あり
- レントゲンは**デジタル化・電子媒体保存**前提（→「電子画像管理加算」「デジタル撮影」で算定）

## 各事例のSOAPカルテ本文（入力・SOAP書き起こし版）

> 上の「カルテ本文（入力）」を、**本プロダクトの実際の入力に近い自由記述SOAP**へ書き起こしたもの。
> これを入力 → 期待抽出（病名・オーダー）→ 期待算定（下の「算定明細」）で、**SOAP→抽出→算定のフルend-to-end**を評価できる。
> 方針：算定対象は**すべて「実施済み」**として記述（予定・依頼として書くと抽出で除外されるため）。薬剤は**名称・用量・用法・日数**を明示。原カルテの病名・診療内容に**忠実**で、S/Oの所見のみ臨床的に自然化。

## 評価(gold)としての使い方

このプロダクトのパイプライン「**カルテ本文 → 病名・オーダー抽出 → 算定**」に対して：

1. **Stage2（オーダー→算定）の回帰／パリティ**
   - 各事例の診療内容を構造化オーダーとして自engineへ入力 → 出力を上の算定明細（区分・名称・組み合わせ）と照合。
   - 点数の絶対値は改定差があるため、**ORCAお試しサーバで同事例を引き、ORCA基準で点数照合**するのが確実（＝正解オラクル）。
   - 既存の `evaluate-gold-*` / gold列（`正解点数`/`正解ステータス`/`正解コード`）形式に各事例を起こせば、既存基盤で評価できる。

2. **Stage1（カルテ本文→抽出）の参照**
   - 本プロダクトの入力は自由記述SOAPだが、本事例は半構造化カルテ。**「実施/予定」「院内/院外」「同一日複数科」「在宅」**など、抽出で判定が要るパターンの**現実的なサンプル**として使える。
   - 原文カルテ（病名＋処方/処置/検査の記述）が必要なら、`scripts/fetch_orca_karte_cases.sh` で原PDFを取得し、`*-print.txt` の上部（診療日時・処方・手術・処置等）を参照。

3. **未対応範囲の確認**
   - 現engineは外来検体検査中心。本事例には**初再診・医学管理・在宅・精神科・病理・画像**が多数含まれるため、「どこまで自engineが算定できるか／落ちるか」の**カバレッジ測定**にも有用。

## 次の一手（推奨）
- まず `scripts/fetch_orca_karte_cases.sh` を実行 → 11事例を取得。
- お試しサーバ（`weborca-trial`）で同事例のレセプトを開き、**自engine vs ORCA** を1事例ずつ照合（無料・即・正解付き）。
- 照合フォーマットを既存 gold（`正解点数`/`正解コード`/`正解ステータス`）に合わせ、`evaluate-gold-*` に載せて自動化。
