# サポート・通知方針（草案）

更新日: 2026-05-06

## 位置づけ

本書は現時点の運用方針を整理した内部草案であり、契約上の保証値を直接定めるものではない。正式な SLA として対外提示する場合は、法務・営業・運用の確認を経る。

## 基本方針

- サポート受付時間: 平日 9:00-17:00（土日祝日を除く）
- 障害・セキュリティ事故は上記時間外でも検知・対応対象となる場合がある
- 契約管理者向けの重要通知はメールで実施する

## 優先度の考え方

- 緊急: ログイン不能、SOAP 生成不能、決済不能、広範な障害
- 高: 一部組織での主要機能障害、初回設定不能、監査上重要な異常
- 通常: 機能改善要望、軽微な UI 不具合、文言修正

## 事故時の通知方針

- 個人情報保護法上の報告対象となり得る事故は、法令と runbook に従って対応する
- 医療機関等への通知は、影響範囲、一次対処、再発防止方針を含める
