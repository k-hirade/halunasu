# 緊急時治療管理の現行単位・実装状況調査

調査日: 2026-07-16  
対象: 介護報酬の「緊急時治療管理」と、名称が似ている医科診療報酬項目  
結論の確度: 現行単位は公式告示・令和8年6月版算定構造・サービスコード表で確認済み

## 1. 結論

2026年7月16日時点における介護報酬の「緊急時治療管理」は、**518単位/日**である。

これは次のサービスで確認できる。

| 制度 | サービス | 上位項目 | 現行単位 |
| --- | --- | --- | ---: |
| 介護保険 | 介護老人保健施設 | 緊急時施設療養費 | 518単位/日 |
| 介護保険 | 介護医療院 | 緊急時施設診療費 | 518単位/日 |
| 介護保険 | 老健の短期入所療養介護 | 緊急時施設療養費 | 518単位/日 |
| 介護保険 | 介護医療院の短期入所療養介護 | 緊急時施設診療費 | 518単位/日 |
| 介護保険 | 上記に対応する介護予防短期入所療養介護 | 各緊急時施設療養費・診療費 | 518単位/日 |

厚生労働省の現行告示は、老健と介護医療院の双方について518単位とし、同一入所者につき月1回、連続3日までとしている。令和8年6月1日版の算定構造でも518単位が維持されている。

ただし、**「緊急時治療管理なら常に518」と扱ってはいけない**。名称が似ている医療保険の「緊急時施設治療管理料」は別項目で、現行500点である。請求制度、サービス種別、算定日を先に確定しなければならない。

## 2. 518、511、500が併存して見える理由

### 2.1 現行の介護報酬は518単位

[指定施設サービス等に要する費用の額の算定に関する基準](https://www.mhlw.go.jp/web/t_doc?dataId=82aa0255&dataType=0)では、次が現行値として掲載されている。

- 介護老人保健施設: 緊急時施設療養費の緊急時治療管理、518単位/日
- 介護医療院: 緊急時施設診療費の緊急時治療管理、518単位/日
- 同一入所者について月1回、連続する3日まで
- 重篤となり救命救急医療が必要な場合に、投薬、検査、注射、処置等を行ったときに算定

[令和8年6月版 介護報酬の算定構造](https://www.mhlw.go.jp/content/001675193.pdf)でも、老健、介護医療院、短期入所、介護予防短期入所について518単位と確認できる。したがって、2026年7月診療・サービス分を500単位又は511単位として扱う根拠はない。

### 2.2 511単位は2019年9月以前の旧額

2019年10月1日適用の[厚生労働省告示第101号](https://www.mhlw.go.jp/content/12404000/01.pdf)は、緊急時治療管理を511単位から518単位へ改定している。よって、511単位は履歴として保持すべき値であり、現在値ではない。

### 2.3 介護の旧通知には500単位という記載が残る

[実施上の留意事項（老企第40号）](https://www.mhlw.go.jp/web/t_doc?dataId=00ta4383&dataType=1&pageNo=2)の該当段落には、現在も「一日につき五〇〇単位」と表示されている。一方、その後の改定告示と現行の単位数表は518単位である。

この不一致は次のように扱う。

1. 単位数は、算定日現在の告示、算定構造、サービスコード表を優先する。
2. 留意事項は、対象状態、実施行為、連続日数、併算定制限等の解釈に使用する。
3. 留意事項に残る500単位を現行価格マスタへ転記しない。

医療事務担当者が518単位に疑問を持ったのは合理的である。厚生労働省サイト内だけでも500と518が併存して見えるため、資料名と適用時期を確認しないと誤読しやすい。

### 2.4 医療保険には現在も500点の別項目がある

[医科診療報酬点数表](https://www.mhlw.go.jp/web/t_doc?dataId=84aa9729&dataType=0)には「緊急時施設治療管理料 500点」がある。これは介護報酬の518単位とは別制度・別要件である。

| 項目 | 介護の緊急時治療管理 | 医科の緊急時施設治療管理料 |
| --- | --- | --- |
| 単位 | 518単位/日 | 500点 |
| 制度 | 介護保険 | 医療保険 |
| 主な対象 | 老健・介護医療院等の入所者 | 所定の転換老健に入所する患者 |
| 主な条件 | 重篤で救命救急医療が必要、施設内で投薬・検査・注射・処置等 | 併設保険医療機関の医師が、夜間又は休日に緊急往診 |
| 回数 | 月1回、連続3日まで | 1日1回、月4回まで |

名称だけで両者をマッピングすることは誤請求につながる。

## 3. 現行サービスコード表の確認

[令和8年6月1日適用の介護サービスコード資料](https://www.wam.go.jp/gyoseiShiryou/detail?ct=020050010&gno=22560)を確認した。対象行は次のとおりで、いずれも518単位である。

| サービスコード | サービスコード名称 | 単位 |
| --- | --- | ---: |
| `529000` | 保健施設緊急時治療管理1 | 518 |
| `526000` | 保健施設緊急時治療管理2 | 518 |
| `556000` | 医療院緊急時治療管理 | 518 |
| `229000` | 老短緊急時治療管理1 | 518 |
| `226000` | 老短緊急時治療管理2 | 518 |
| `2A6000` | 医療院短期緊急時治療管理 | 518 |

実装では表示名「緊急時治療管理」だけからコードを決めず、施設のサービス種別と算定日に有効なコードを解決する必要がある。

## 4. 算定要件の確認

現行告示と留意事項から、少なくとも次を決定論的に確認する必要がある。

1. 対象の介護サービスを利用している。
2. 入所者の病状が重篤で、救命救急医療が必要な状態である。
3. 緊急的な治療管理として投薬、検査、注射、処置等を実施している。
4. 同一入所者について当月1回目である。
5. 同一エピソードの連続3日以内である。
6. 緊急時治療管理と特定治療を同時算定していない。
7. 所定疾患施設療養費等、サービス別の併算定制限に抵触しない。

留意事項が挙げる対象状態は次の6区分である。

1. 意識障害又は昏睡
2. 急性呼吸不全又は慢性呼吸不全の急性増悪
3. 急性心不全（心筋梗塞を含む）
4. ショック
5. 重篤な代謝障害
6. その他薬物中毒等で重篤なもの

発熱、転倒、嘔吐などの事象名だけでは算定確定できない。6区分のいずれに該当し、救命救急医療が必要だったか、何を実施したかの記録が必要である。

## 5. 西山病院への適用

西山病院グループの[施設紹介](https://www.nishiyama.or.jp/facility/)と[受入対象者資料](https://www.nishiyama.or.jp/files/ukeire.pdf)から、グループ内には医療療養、介護医療院、介護老人保健施設、短期入所が存在することを確認した。

- 西山病院の東館・南館: 介護医療院として案内されている。
- 西山ナーシング: 介護医療院として案内されている。
- 西山ウエルケア: 介護老人保健施設として案内されている。
- 短期入所療養介護の提供も案内されている。

したがって、西山病院グループで518単位が使われる可能性は高く、制度上も整合する。ただし、提供された確認票の画像だけでは、どの施設、病棟、サービス、事業所番号の請求に使う票かを特定できない。

西山病院へ次の1点を確認すれば、適用コードまで確定できる。

> この確認票は、介護医療院、介護老人保健施設、短期入所療養介護のどの事業所・サービスの請求に使用していますか。また、現在の請求システムに表示されるサービスコード名称とコードは何ですか。

医療療養病床又は医療保険請求の票であれば、介護の518単位を使用してはならない。

## 6. 提供画像の評価

提供された画像のうち、説明資料には518単位、月1回、連続3日までという記載があり、現行告示と整合する。院内確認票には6つの対象状態、急変時病名、処置、検査、処方、急変状況を記録する欄があり、算定点検の入力票として有用である。

ただし、提供画像で確認できる範囲では次が不足している。

- 保険制度
- 施設・事業所・サービス種別
- サービスコード
- 対象日と連続エピソード
- 当月既算定の有無
- 併算定項目
- 医師判断・確認者・確認日時

画像はワークフロー要件の根拠には使えるが、単位数マスタの根拠には使わない。

## 8. 現時点の安全な製品挙動

介護機能を実装するまでは次を守る。

1. 現行の医科算定画面に介護518単位を表示しない。
2. 介護の確認票や記録を医科コード`180709110`へ自動変換しない。
3. 入力が介護保険由来と判明した場合は、「介護報酬のため現在の医科算定対象外」と明示する。
4. 518をソースコードへ固定値として埋め込まない。
5. 実装時は`billingScheme + careServiceType + serviceDate`から有効なサービスコードと単位を取得する。

## 9. 実装優先順位

### P0: 制度混同防止とマスタ

- `billingScheme`、`careServiceType`、`serviceDate`を必須化
- effective-datedな介護サービスコードマスタを追加
- 医科点数と介護単位を別集計にする
- 医科`180709110`への誤マッピングを禁止

### P1: 緊急時治療管理の点検

- 急変エピソードと日別実施記録
- 6状態、救命救急判断、実施事実の確認
- 月1回、連続3日、併算定制限の決定論点検
- 人による確認後にのみ算定候補化
- 西山病院確認票に対応した入力・印刷

### P2: 月次請求

- 入所者・事業所・請求月単位の集計
- 地域区分とサービス別単価による金額計算
- 介護給付費請求データの作成・検証
- 改定マスタ取込と世代間回帰テスト

## 10. 根拠の優先順位

今後の改定調査では、次の順序で判定する。

1. 算定日現在の厚生労働省告示・単位数表
2. 同じ適用日の介護報酬算定構造・介護サービスコード表
3. 実施上の留意事項、Q&A、疑義解釈
4. 施設の請求システム表示と請求実績
5. 院内様式、解説記事、二次資料

単位数を確定するときは1と2を必須とし、旧通知や院内資料だけでは更新しない。
