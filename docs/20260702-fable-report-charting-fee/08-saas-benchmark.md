# 08. 他SaaSベンチマーク（追加テーマ）

2026年時点の日本の医療AI／レセプト点検SaaS市場と、本プロダクト（fee＝算定/点検, charting＝AIカルテ）を比較する。比較のために自由に立てた追加テーマ。

## 市場マップ（2026）

### AIカルテ／診療記録（charting の競合）

| サービス | 立ち位置 | 本プロダクトとの関係 |
|---------|---------|--------------------|
| **ユビー生成AI** | 会話をリアルタイム文字起こし→カルテ整形。診療記録・退院サマリ・紹介状取り込み。病院向け個別見積 | charting のアンビエント録音→SOAP と正面から競合。ユビーは病院規模・診療科数での価格設計 |
| **medimo** | 医療用語特化の自社STT。疾患名・薬剤名を反復学習し高精度文字起こし | charting の逐語精度で比較される。medimo は「医療特化STT」を差別化に。→ [07](07-compliance-phi-ops.md) の国内・特化STT検討の参照先 |
| **エムスリーデジカル / カルテZERO 等** | AI搭載クラウド電子カルテ | 電子カルテ本体。charting は「記録支援」レイヤーとして共存も競合もしうる |

2026年改定で「AIを活用した業務効率化」が診療報酬の評価対象に加わった点は、両プロダクトの追い風。

## 本プロダクトの差別化ポイント（強み）

1. **カルテ→算定→点検の一気通貫**: charting（記録）と fee（算定/点検）を同一プラットフォームで持つ。ユビーやMightyCheckerは片側に寄る。**「記録の質が算定の質を決める」ことを一社で最適化できる**のは構造的な強み。
2. **決定的算定エンジン＋LLM前処理の切り分け**: 点数計算を正典Python（監査可能・再現可能）に置き、LLMは臨床イベント抽出に限定。「AIが勝手に点数を作らない」設計は、医事の信頼と説明責任に効く。競合の"生成AI丸投げ"より監査適合性が高い。
3. **差分診断が baseline を正解と扱わない**: 既存レセコンの結果を鵜呑みにせず全件を要確認に落とす姿勢は、導入時の「まず現状把握」ニーズ（既存レセとの突合）に的確。
4. **導入前一括診断（STG）**: レセコン移行前に既存UKE/CSVと突合できる。競合の多くは「導入後に効く」ので、**導入意思決定を後押しする材料**として独自。

## 戦略的提言（プロダクト観点）

1. **「一気通貫」を前面に**: 記録の質→算定精度の因果を可視化する機能（例: このSOAP記載があればこの算定が根拠付く、という往復リンク）。競合が持てない体験。
2. **「監査可能なAI算定」をブランドの軸に**: 支払基金AI審査時代に、「なぜこの点数か」を根拠付きで即提示できることは強い差別化。差分診断の理由表示を各所へ展開。
3. **点検ルールDBの継続拡充**をロードマップに明示。網羅性は導入判断の主要因。
4. **導入前診断を営業武器に**: STG限定の一括診断を「無料アセスメント」的に使い、既存レセの取りこぼしを可視化して乗り換え動機を作る。
