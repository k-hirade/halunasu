# 05. アーキテクチャ妥当性評価と汎用精度改善計画（反例防止つき）

作成日: 2026-07-04
目的: 「特定カルテの修正」でなく**汎用的に**精度を上げ、**反例（新たな誤算定・回帰）が生まれない**構造を作る。そのために現アーキテクチャが「すべてのカルテを正しく算定できるか」を評価する。

---

## 1. 調査でわかった事実（実データ・実コード）

### 1.1 決定論エンジンは健全（最重要の事実）
- **オフライン評価（抽出をルール/CCで固定）で 138ケース exact 100% pass**（`v2-rebaseline-20260615-exact-cc.json`、avg 61ms）。
- 電子点数表ルール（**包括・背反・回数上限・必須コメント**）は `electronic_rules.py` として実装され算定に適用済み。
- gold差分の分類→改善バックログ→アクションプラン化の機構（`claim_batch.py` の gold_* 群）まで既にある。
- → **誤差の主要源は決定論エンジンではなく、(a) LLM抽出層、(b) 施設文脈依存のコード選択、(c) gold自体の陳腐化**（物価対応料）に局在する。STG実測（04参照）とも整合：CT時制誤分類・院内処方漏れ・run間変動が失敗の中身だった。

### 1.2 goldデータセットの分布は偏っている（「すべてのカルテ」への反例源）
391ケースの構成: IM 172 / PED 58 / DERM 53 / ENT 51 / OPH 24 が大半、種類は **LAB系 ~187（過半）**。
一方 **SURG 11・PATH 4・ENDO 4・TIME(時間外) 4・HOME 2・リハ/透析/精神はほぼ無し**。
→ この分布の外（手術・麻酔・リハ・在宅・精神・透析・入院/DPC）では**評価されていない＝反例が潜在**。

### 1.3 実世界の査定分布との整合（Web調査）
査定の主因は **A(適応外・病名漏れ)・B(過剰)・D(算定要件不合致)・縦覧(◯月に1回)**。外来では**薬剤過剰・検査過剰・D査定**が多い（[ソラスト](https://www.solasto.co.jp/solastoonline/iryou/receipt-assessment/)、[病院経営事例集](https://hpcase.jp/sugai_medical_fees006/)、[NEC](https://www.nec-nexs.com/supple/medical/column/tanaka/column054.html)）。外来の査定率は~0.5%・返戻率~0.6%が実務水準（[公開病院データ](https://hospital-miyoshi.jp/wp-content/uploads/2024/11/d7d0ac5e1763264895b3b076dfb43834.pdf)）。
→ 我々の点検資産（適応/禁忌/回数/縦覧）は実世界のリスク分布と**方向が一致**。強化すべきは「B過剰・D要件・縦覧」の決定論判定。

---

## 2. アーキテクチャ評価（結論）

**骨格（LLM=抽出のみ／決定論=算定／HITL）は適切。** ただし「すべてのカルテを正しく」には、次の3つの**構造的前提**を明示的に設計に組み込む必要がある：

| 前提 | 意味 | 現状 | 必要な強化 |
|---|---|---|---|
| **P-A: LLMは確率的** | 同一カルテでも抽出が揺れる（実測: 同一ケースで7→12件）。LLM単独に頼る限り反例はゼロにならない | checklist recall・決定論candidatesで部分的に補完 | **決定論の回収網を広げる**（LLMが落としても決定論が拾う）＝汎用改善の本線 |
| **P-B: コード選択は施設文脈依存** | CT列数・撮影装置・施設基準で正解コードが変わる（カルテだけでは決まらない） | facility_imaging_profile / facility_standard_keys あり（known gap 1件） | 施設プロファイルの網羅拡大＋**施設基準由来の加算はLLM非依存で決定論導出**（例: 検体検査管理加算） |
| **P-C: 対応範囲には境界がある** | 入院/DPC・手術詳細・公費按分などは現スコープ外 | unsupported domain の抽出保持はある | **範囲外を確実に needs_review へ落とす**（誤算定でなく要確認＝**反例の安全化**）。「全カルテを正しく算定」ではなく「範囲内は正しく、範囲外は必ず人へ」が正しい目標設定 |

---

## 3. 計画（4フェーズ）

### P1: 計測と守りの確立（反例を"検出できる"状態に）
1. **フルgold STGラン**（391件 or 層化サンプル100件）で**実LLM込みの真のpass率・失敗分布**を測る（現状5-10件runのみ）。`--repeat 3` で **stability（同一ケース再現率）** を初計測。
2. **CIゲート**: 決定論gold（オフライン138+）は毎PR必須。LLM系E2Eは**プロンプト版変更時に必須**。SafetyRatio（過大/過小の非対称）を採点に追加。
3. **gold陳腐化の解消**: 物価対応料（180820010/180819910）を期待値へ反映（+2系の見かけfailを一掃）。
4. 本番/STGの **shadow calculation 差分監視**を反例回収ループにする（新プロンプト/新ルールの影響を本番前に観測）。

### P2: 汎用精度＝決定論の網を広げる（特定ケース修正ではなく構造で拾う）
1. **決定論リカバリの拡大**: `deterministicClinicalCandidates`/checklist recall の対象を、実測で漏れた**院内処方（調剤料/処方料）・CT/画像・時間外**へ拡張。「カルテに『CT』の字があるのにimagingイベントが無い」→ 決定論で needs_review 候補を復元（LLM揺れの下限を保証）。
2. **施設基準由来加算の完全決定論化**: 検体検査管理加算等（run間で出たり消えたりした）はLLM出力に依存させず、施設プロファイル＋算定内容から常に導出。
3. **範囲外検出の強化（P-C）**: 入院・DPC・手術詳記等のシグナルを検知したら算定を止めず**必ず needs_review**（unsupported理由つき）。反例が「誤った確定算定」になる経路を塞ぐ。
4. **B過剰・縦覧の決定論判定強化**: electronic_rules の回数/背反に**患者履歴（縦覧窓）**を接続（recept-checker移植資産）。実世界の査定主因に直結。

### P3: LLM層の安定化（揺れの抑制）
1. **確信度較正＋3値昇格**（certainty=ambiguous/inferred → 自動採用させない）。
2. **高額イベントのみ二重抽出/self-consistency**（1000点超・査定頻発コード限定でコスト最小）。
3. **時制/主体の反例スイート**: CT past_or_external型の誤分類パターンを合成カルテで体系化し（前回/他院/持参/予定 × 画像/処置/処方のマトリクス）、プロンプト変更ゲートに組み込む＝**同型の反例の再発防止**。

### P4: 分布の拡大（未評価領域の反例を先に潰す）
1. **goldの薄い領域へケース追加**: SURG/ENDO/PATH/TIME/HOME/リハ/精神。既存の合成生成器（`generate:fee-soap-e2e:coverage-800`）を活用し、まず各領域L1/L2を10件ずつ。
2. **実カルテ由来の匿名ケース取り込み**: 西山PoC（P-1匿名化ツール）で得る実データを、匿名化→gold化する運用ルートを定義（合成の文体限界を補う）。
3. **メタモルフィックテスト**: 「言い換え・行順入替・ノイズ追記でも算定結果は不変」「イベント追記で総点数は減らない」等の**性質テスト**で、gold外の反例を自動探索。

---

## 4. 優先順（推奨着手）

1. **P1-1 フルgold STGラン＋stability計測**（現在地を知らずに汎用改善は語れない。1回の実行で失敗分布が層別に見える）
2. **P1-3 gold更新（物価対応料）**＋ **P2-2 施設基準加算の決定論化**（見かけノイズと実揺れを同時に除去→以後の計測が綺麗になる）
3. **P2-1 決定論リカバリ拡大**（CT/院内処方/時間外）— 実測で確認済みの漏れを「構造」で塞ぐ
4. **P1-2 CIゲート** → 以後のすべての変更に反例防止が効く
5. P3 → P4 の順で拡張

## 5. 目標の言語化（提案）

> **「範囲内のカルテは決定論で正しく算定し、範囲外・曖昧は必ず人に渡す。LLMの揺れは決定論の網で下限保証し、すべての変更はgold＋性質テストのゲートを通る。」**

「全カルテを常に正しく算定」は（審査側ですら医学的判断で個別裁量がある以上）到達不能な目標であり、**反例をゼロにするのではなく「反例が誤確定にならない」構造**が正しい安全設計。これは支払基金自身のコンピュータチェックの位置づけとも一致する。
