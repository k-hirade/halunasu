# 患者1013 カルテから月次レセプト比較 E2E

## 結論

合成患者1013のカルテ2件を使い、STGでカルテから月次UKE比較までを3回実行した。処理経路は完走したが、既存UKEの6コード・2,438点に対し、当社結果は423点、423点、371点で、既存コードとの一致は3回とも0件だった。

患者1006と異なり、候補明細と月次点数も安定していない。外来管理加算52点が1・2回目には入り、3回目には入らなかったことが差である。

また、カルテ本文に「傷病手当金意見書を作成・交付」「療養費同意書を交付」と明記されているのに、対応する既存UKEの2コードを候補にも確認事項にも出せなかった。これは施設基準不足ではなく、文書交付イベントから診療行為へ接続する抽出・知識ルールが未実装であることによる明確な機能不足である。

## 評価

### 文書交付料は直接改善できる

2種類の文書は実施事実がカルテに明記されている。リポジトリ内を確認したところ、現行のカルテ算定知識には両文書を候補化するルールがない。文書種別、作成・交付の実施、交付日を正規化し、決定論的にマスターへ接続することで改善できる。

### 薬剤総合評価調整管理料は追加データが必要

カルテには眠気・ふらつきを理由に2剤を中止したことが記載されているため、管理料の検討シグナルとしては拾うべきである。ただし、最初の受診時点で変更前薬剤数が明記されず、処方データも今回の算定入力へ渡していない。自動算定ではなく、変更前後の薬剤数と対象薬剤を求める確認候補が妥当である。

### 外来系候補は在宅区分不足の副作用

毎回、既存UKEにない`難病外来指導管理料の確認`を2受診とも表示した。さらに外来管理加算が確率的に加算された。現行契約では訪問診療を`setting`で表現できず外来として処理されるため、在宅区分を追加したうえで外来専用管理料を排除する必要がある。

## 安定性

月次結果は423・423・371点、確認事項は2・3・1件、抽出イベント数は `2/2`、`4/3`、`5/3` だった。外来管理加算の有無がOpenAI抽出の揺れに影響され、最終点数まで52点変動している。

一方、難病外来指導管理料の確認候補は3回とも各受診に1件ずつ、合計2件出た。候補自体は安定しているが、在宅患者に対する適用可否を先に判定する必要がある。
